### Проблема #1: Тесты падают из-за неправильных ID книг
**Дата:** 2026-06-24
**Статус:**  Решено

**Описание проблемы:**
При запуске тестов ID книг начинаются не с 1, а с больших чисел (34, 37, 41). Это происходит потому, что `Book._last_id` сохраняет значение между тестами.

**Решение:**
Добавлен `conftest.py` файл с фикстурой для сброса `Book._last_id` перед каждым тестом:

```python
import pytest
from src.models import Book

@pytest.fixture(autouse=True)
def reset_book_ids():
    Book._last_id = 0
    yield
    Book._last_id = 0
Результат:
Все 43 теста проходят успешно.
